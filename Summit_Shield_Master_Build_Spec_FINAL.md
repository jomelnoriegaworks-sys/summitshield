# Summit Shield Insurance --- Master Build Specification (Final)

## Project Overview

Build a premium, production-quality website for **Summit Shield
Insurance**, a fictional insurance agency created as a portfolio case
study.

Treat this exactly like a real client project.

------------------------------------------------------------------------

# Primary Objective

Create a **3-page marketing website** that integrates seamlessly with
**GoHighLevel (GHL)**.

The site should demonstrate: - Professional UI/UX - Clean code -
Conversion-focused design - Real-world CRM integration -
Production-ready quality

SEO is **not** a priority.

------------------------------------------------------------------------

# Top Priority: GoHighLevel Compatibility

Every decision must prioritize compatibility with GoHighLevel.

## Requirements

-   HTML must work inside GHL Custom HTML elements.
-   CSS must work inside a single GHL Custom CSS block.
-   JavaScript must be vanilla JS.
-   No React, Next.js, Vue, Angular, Tailwind runtime, or build tools.
-   Avoid dependencies.
-   Use semantic HTML.
-   Keep CSS modular and well commented.
-   Keep JS lightweight.

The final output should require little to no modification before being
pasted into GoHighLevel.

------------------------------------------------------------------------

# Success Criteria

When complete, the project should:

-   Look like a real insurance company's website.
-   Feel premium and trustworthy.
-   Be fully responsive.
-   Be easy to integrate into GoHighLevel.
-   Be suitable as a professional portfolio case study.
-   Be maintainable and well documented.

Think like a senior frontend engineer building a website for a paying
insurance client---not a prototype.
